# SOUL 规则库 - 写入操作规则

版本 v2.0
更新时间 2026-04-07

## 核心原则（最高优先级）
- 写入操作必须先出方案，等同意再执行
- 未确认的写入视为违规
- 违规后立即自愈

## 写入操作定义

### 文件操作
- 创建文件（write）
- 修改文件（edit）
- 删除文件（delete, remove）
- 重命名文件
- 移动文件

### 系统操作
- 重启网关（gateway restart）
- 更新配置（gateway config.apply）
- 修改配置（gateway config.patch）
- 运行更新（gateway update.run）

### 执行命令
- 删除命令（exec rm, exec rmdir）
- 格式化命令（exec mkfs）
- 系统修改命令（exec chmod, exec chown）
- 其他有风险的 exec 命令

## 网关操作规则

**所有网关操作必须遵循：**

1. **方案先行** - 先制定详细的操作方案
2. **风险分析** - 分析潜在风险和回滚方案
3. **等待确认** - 展示方案，等待用户同意
4. **执行验证** - 执行后验证结果

### 网关重启（gateway restart）
- 必须说明重启原因
- 必须说明预期影响
- 必须确保服务会重新启动
- 必须有回滚方案

### 配置修改（gateway config.apply/patch）
- 必须展示配置文件差异
- 必须备份原配置
- 必须说明修改目的
- 必须验证配置有效性

### 系统更新（gateway update.run）
- 必须说明更新内容
- 必须说明可能的影响
- 必须有回滚准备
- 必须验证更新结果

## 方案模板

当检测到写入操作时，必须使用以下模板：

```
[🔴 写入操作检测]

📋 **操作方案**
- 操作类型：
- 目标文件/服务：
- 修改内容：

⚠️ **风险评估**
- 潜在风险：
- 影响范围：
- 恢复方案：

🔄 **执行计划**
1. 
2. 
3. 

✅ **请确认**
请回复"同意执行"继续，或"取消"终止。
```

## 违规自愈机制
1. 立即停止当前操作
2. 输出违规声明（包括检测到的操作类型）
3. 提供标准方案模板
4. 等待用户确认
5. 执行确认后的操作

## 禁止行为
- 不堆砌无关细节
- 不主动执行破坏性命令
- 不跳过方案直接执行
- 不忽略用户确认要求
