## 任务分类触发条件

| 级别 | 触发条件 | 关键词示例 |
|------|----------|------------|
| **L0** | 一步问答、查时间、简单计算 | 「现在几点了」「今天是星期几」「帮我算一下」 |
| **L1** | 读文件、单一搜索、单一 exec | 「看看这个文件」「搜索 XXX」「执行 ls 命令」 |
| **L2** | 分析、推理、创新、探索、验证 | 「分析一下」「为什么」「帮我思考」「验证」 |
| **L3** | 方案生成、创建/修改/删除请求 | 「帮我做」「创建 X」「修改 Y」「删除 Z」 |
| **L4** | 用户确认执行已有方案 | 「同意」「确认」「执行」「开始吧」 |

**简化判断：** 不能明确归类为 L0/L1/L2 的任务 → L3

---

## L3 vs L4 关键区别

| 维度 | L3 方案生成 | L4 方案执行 |
|------|------------|------------|
| 触发 | 用户请求创建/修改/删除 | 用户确认已有方案 |
| 产物 | execution_plan.md | 代码/配置/系统变更 |
| 权限 | 无写入权限 | 有执行认证 |
| 技能 | deep-thinking + openclaw-behavior-plan | planning-with-files + agent-pool |

## 写入操作触发条件

| 触发条件 | 动作 |
|----------|------|
| 涉及文件创建 | 必须先出方案（L3），等确认（L4） |
| 涉及文件修改 | 必须先出方案（L3），等确认（L4） |
| 涉及文件删除 | 必须先出方案（L3），等确认（L4） |
| 涉及代码修改 | 必须先出方案（L3），等确认（L4） |
| 涉及配置修改 | 必须先出方案（L3），等确认（L4） |

**规划文件豁免：** `execution_plan.md`、`*方案*.md` 等可在 L3 阶段直接写入

**违规响应：** 立即停止，输出违规声明，提供方案模板

## 技能触发条件

### 自动触发（用户直接指定）
- 用户提到具体技能名称 → 直接执行

### 识别后触发（用户提出任务）
- 用户提出任务 → 识别技能 → 制定方案 → 等批准 → 执行

### 特殊技能规则

| 技能类型 | 规则 |
|----------|------|
| 安装新技能 | 必须申请 |
| 创建新技能 | 必须申请 |
| 危险技能 | 一律拒绝 |

### 白名单技能

以下技能执行时跳过认证检查：
- `workflow-manager`
- `agent-pool`
- `planning-with-files`

## 工作流触发条件

**强制检查时机：** 所有任务执行前

**触发条件：**
- 任务需要执行操作

**检查流程：**
1. 扫描 `workflows/` 目录
2. 读取 `_index.yaml`
3. 匹配工作流节点
4. 找到 → 调用 `workflow-manager` 强制执行
5. 未找到 → 进入普通 L3 流程

## 四层拦截触发

### Layer 1: 技能白名单
- 白名单技能的所有操作 → 直接放行

### Layer 2: 破坏性命令（永久禁止）
| 类别 | 命令 |
|------|------|
| 磁盘破坏 | `dd if=`, `mkfs`, `format` |
| 系统破坏 | `> /etc/passwd`, `> /boot/` |
| Fork 炸弹 | `:(){:|:&};:` |
| 远程执行 | `\| bash`, `\| sh`, `\| sudo` |
| 根目录删除 | `rm -rf /` |

**即使有执行认证也无法执行。**

### Layer 3: 增删改操作（需要执行认证）
| 类别 | 命令/工具 |
|------|----------|
| 文件操作 | `rm`, `mv`, `cp`, `mkdir`, `chmod` |
| Git 操作 | `git push`, `git commit`, `git add` |
| 包管理 | `pip install`, `npm install`, `apt install` |
| 系统服务 | `systemctl start/stop`, `docker run/rm` |
| 工具 | `write_file`, `patch` |

**需要用户确认后授予执行认证。**

### Layer 4: 其他操作
- 所有查询类命令 → 直接放行

## Agent Pool 并发控制触发

| 条件 | 动作 |
|------|------|
| ≤3 个任务 | 直接并行执行 |
| >3 个任务 | 分批执行，等待上一批完成 |
| 任意时刻 | 最多 3 个并行 |

---

## 错误恢复触发

| 触发条件 | 响应 |
|----------|------|
| 第一次违反约束 | 输出违规声明，重新执行 |
| 第二次违反（同一对话） | 停止，请求指导 |

## 自我改进触发

| 时机 | 动作 |
