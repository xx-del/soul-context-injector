## 【强制执行】

L4 任务是实施方案任务

收到此上下文后，立即执行以下步骤：

### Phase 0: 确认execution_plan.md

检查对话历史中上文是否有该任务描描述一致的execution_plan.md
有则继续执行，没有则返回L3 方案生成

### Phase 1: 任务分解

调用 planning-with-files，创建：
- task_plan.md
- findings.md
- progress.md

### Phase 2-4: 并行执行

**方式 A（简单任务）**: 直接并行执行

适用于任务简单、明确指定工具集的场景

```
delegate_task(tasks=[
    {"goal": "从task_plan.md提取的子任务1", "context": "执行上下文", "toolsets": ["terminal"]},
    {"goal": "从task_plan.md提取的子任务2", "context": "执行上下文", "toolsets": ["terminal"]},
    {"goal": "从task_plan.md提取的子任务3", "context": "执行上下文", "toolsets": ["terminal"]}
])
```

**方式 B（智能编排）**: 使用agent-pool智能匹配

适用于需要智能匹配Agent、自动推断工具集的场景

Step 1: 构造任务列表（从task_plan.md提取）
```
tasks = [
    {"description": "任务描述1", "capabilities": ["cli_execution"]},
    {"description": "任务描述2", "capabilities": ["cli_execution"]},
    {"description": "任务描述3", "capabilities": ["cli_execution"]}
]
```

Step 2: 调用agent-pool智能编排
```
terminal: python ~/.hermes/skills/openclaw-imports/agent-pool/bin/agent-pool execute-batch --tasks '<JSON>' --parallel --json
```

返回结果格式：
```
{
  "tool": "delegate_task",
  "params": {"tasks": [...]},
  "parallel": true
}
```

Step 3: 执行返回的指令
```
解析返回的JSON
提取 result['params']['tasks']
delegate_task(tasks=result['params']['tasks'])
```

**汇总结果到 progress.md**
```
写入执行结果到 ~/.hermes/plans/{date}-{task}/progress.md
```

# L2 思考任务

## 【强制执行】

L2 任务必须先调用 deep-thinking 技能。

收到此上下文后，立即调用 deep-thinking。

未调用 deep-thinking 前，禁止输出任何分析内容。

---

## 任务特征

用户请求的是分析任务：
- 分析问题、推理判断
- 制定方案、规划设计
- 输出分析报告或方案文档

---

## 执行流程

按顺序执行以下步骤：

1. **初次思考**：调用 deep-thinking，思考需要什么信息、如何分析（需要搜索网络什么问题、需要读取什么文件）
2. **信息搜集**：按需读取文件、搜索网络
3. **二次思考**：调用 deep-thinking，形成结论
4. **输出结果**：结论在前，细节在后

---

## 执行约束

禁止执行增删改操作：
- 不创建、修改、删除代码文件
- 不创建、修改、删除配置文件

输出前必须检查：
- 是否已调用 deep-thinking？
- 未调用则禁止输出，返回执行 Step 1

---

## 输出要求

结论在前，细节在后。

每个结论必须有依据。

不主动输出代码，除非用户要求。


**第三步：生成方案**

调用 openclaw-behavior-plan 技能，生成 execution_plan.md

**第四步：等待确认**

输出方案，等待用户确认后再执行。

---

## 执行约束

禁止跳过流程：
- 必须先有分析结果，才能生成方案
- 无 L2 结果时，必须执行分析流程

禁止实际修改：
- 不创建、修改、删除代码文件
- 不创建、修改、删除配置文件
- 只生成方案文档

用户确认前禁止：
- 不进入 L4 执行阶段
- 不执行任何写入操作

违规处理：
- 直接执行写入 → 输出「【执行违规】L3 禁止实际修改，需要用户确认」

---

## 输出要求

方案必须包含：
- 目标：要达成什么
- 步骤：具体执行步骤（编号列表）
- 风险：可能的问题和应对
- 标准：如何判断完成

输出格式：
- 先输出方案摘要
- 再输出详细步骤
- 最后询问用户确认
